# Спецификация приложения для изучения английского языка (English Learning App)

## 1. Обзор проекта
**Название:** English Master (рабочее название)  
**Цель:** Создание веб-приложения для самостоятельного изучения английского языка с использованием интерактивных карточек, отслеживанием прогресса и адаптацией под уровень пользователя.  
**Платформа:** Responsive Web Application (адаптивный веб-сайт).  
**Основной стек:** React (Frontend), Backend API + Database (Backend).

---

## 2. Технический стек

### 2.1. Frontend
- **Библиотека:** React (версия 18+).
- **Язык:** TypeScript (рекомендуется для типизации данных).
- **Стилизация:** Tailwind CSS или Styled Components (для быстрой адаптивной верстки).
- **Управление состоянием:** Zustand или React Context API (для хранения состояния сессии и пользователя).
- **Роутинг:** React Router DOM.
- **Анимации:** Framer Motion (для переворачивания карточек и переходов).

### 2.2. Backend
- **Среда:** Node.js (Express/NestJS) или Python (FastAPI/Django).
- **База данных:** PostgreSQL (реляционная структура удобна для связей Пользователь-Прогресс) или MongoDB.
- **Аутентификация:** JWT (JSON Web Tokens) для сохранения сессии и синхронизации между устройствами.

### 2.3. Инфраструктура
- **Хостинг:** Vercel/Netlify (Frontend), Heroku/Render/AWS (Backend).
- **Хранение данных:** Облачная база данных (Supabase, Neon, или MongoDB Atlas).

---

## 3. Функциональные требования

### 3.1. Аутентификация и Профиль
- Регистрация/Вход (Email + Password).
- Профиль пользователя (имя, текущий уровень, общая статистика).
- Синхронизация прогресса между устройствами через базу данных.

### 3.2. Выбор уровня сложности (Ручной режим)
- Пользователь выбирает уровень перед началом занятия:
  - **Beginner** (A1-A2)
  - **Intermediate** (B1-B2)
  - **Advanced** (C1-C2)
- Фильтрация контента (слов/предложений) происходит на основе выбранного уровня.

### 3.3. Темы и Категории
- Слова и предложения разбиты по темам (например: "Путешествия", "Бизнес", "Еда", "Грамматика").
- Возможность выбора конкретной темы для занятия или режим "Микс".

### 3.4. Режим обучения (Карточки)
- **Типы вопросов:**
  1.  **Multiple Choice (Выбор варианта):** Вопрос + 4 варианта ответа.
  2.  **Construction (Составление):** Перемешанные слова, нужно собрать предложение/фразу в правильном порядке.
- **Интерфейс карточки:**
  - Лицевая сторона: Вопрос / Задание.
  - Действие: Клик по карточке переворачивает её (Flip animation).
  - Обратная сторона: Правильный ответ + Пояснение (если есть).
- **Обратная связь (после переворота):**
  - Появляются две кнопки: **"Правильно"** (Green) и **"Ошибка"** (Red).
  - Нажатие фиксирует результат в базе данных.
  - Автоматический переход к следующей карточке после выбора.

### 3.5. Трекинг ошибок
- Система запоминает вопросы, где пользователь нажал "Ошибка".
- Пометка вопроса в базе данных как `requires_review`.
- Возможность фильтрации в будущем (режим "Работа над ошибками").

### 3.6. Статистика
- Отдельная страница `/statistics`.
- **Метрики:**
  - Всего вопросов пройдено.
  - Количество правильных ответов.
  - Количество неправильных ответов.
  - Процент успеха (Accuracy %).
  - Прогресс по уровням (сколько слов выучено на каждом уровне).
- Визуализация: Графики или прогресс-бары.

### 3.7. Контент
- В базе данных изначально заложен базовый словарь (Seed Data).
- Структура контента поддерживает расширение (админ-панель не требуется в MVP, но структура БД должна позволять добавление новых слов через скрипты или SQL).

## 4. Структура данных (Модель БД)

### 4.1. Users (Пользователи)
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный ID |
| email | String | Email для входа |
| password_hash | String | Хеш пароля |
| created_at | Timestamp | Дата регистрации |

### 4.2. Topics (Темы)
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный ID |
| title | String | Название темы (напр. "Food") |
| level | Enum | Beginner, Intermediate, Advanced |

### 4.3. Questions (Вопросы/Слова)
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный ID |
| topic_id | FK | Ссылка на тему |
| type | Enum | 'multiple_choice', 'construction' |
| question_text | String | Текст вопроса или слово |
| correct_answer | String | Правильный ответ |
| options | JSON | Варианты ответов (для multiple_choice) или слова для сборки (для construction) |
| level | Enum | Уровень сложности |

### 4.4. UserProgress (Прогресс пользователя)
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный ID |
| user_id | FK | Ссылка на пользователя |
| question_id | FK | Ссылка на вопрос |
| is_correct | Boolean | Результат последнего ответа |
| attempts | Integer | Сколько раз пытались ответить |
| last_reviewed | Timestamp | Дата последнего просмотра |
| status | Enum | 'new', 'learning', 'mastered', 'error' |
